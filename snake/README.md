### Общие сведения

В рамках выполнения примера, внес изменения в ряд файлов оригинальног проекта. Все измененные файлы приведены в данном репозитории. 

### Описание изменений
1. Изменены файлы оригинального  GameField, чтобы он соответствовал реализации  как 2-тора. Изменения коснулись [метода create_position](https://github.com/a-max-22/ht/blob/7dc402f172f1121621c3fec2b109745fae25c09f/snake/game_field.py#L26). Суть изменений состоит в том, что координаты позиции  на поле генерируются  представляются через обходов  по каждой из базовых петель 2-тора. Для обеспечения совместимости, Position так же представляется как Prduct двух целых чисел.
2. Реализовано [вычисление кратчайшего пути от головы до еды](https://github.com/a-max-22/ht/blob/7dc402f172f1121621c3fec2b109745fae25c09f/snake/demo_mode.py#L102), которое применяется в демо-режиме автоматической игры без игрока.
3. Реализован [демо-режим автоматической игры](https://github.com/a-max-22/ht/blob/main/snake/demo_mode.py).
   -  автомтически определяется путь до еды, участки которого [представляются через Path](https://github.com/a-max-22/ht/blob/7dc402f172f1121621c3fec2b109745fae25c09f/snake/demo_mode.py#L117)
   -  режим игры (демо или с игроком) [определяется через Пи-тип](https://github.com/a-max-22/ht/blob/7dc402f172f1121621c3fec2b109745fae25c09f/snake/game.py#L153)
   -  расшируено [состояние GameState](https://github.com/a-max-22/ht/blob/7dc402f172f1121621c3fec2b109745fae25c09f/snake/game_data_structures.py#L128)
   -  в рамках реализации демо-режима inputHandler не модифицировал, так как поначалу посчитал, что будет более естественным, если все расчеты и поведение бота не будут влиять на ранее созданные компоненты игры.

Для упрощения, в рамках демо-режима не работают никакие нажати клавиш (в целом, наверное нужно, чтобы работало их какое-то ограниченное количество, например клавиша выхода).
